## 4 Bước

### Bước 1: SURVEY
Chạy Sean Ellis survey (xem `sean_ellis_survey.md`)

### Bước 2: SEGMENT — Tìm High-Expectation Customers (HXC)

```
TỪ SURVEY DATA:

"Rất thất vọng" → HXC (yêu sản phẩm)
"Hơi thất vọng" → On-the-fence (gần yêu)
"Không thất vọng" → NOT target (bỏ qua)
```

**Tìm pattern trong HXC:**
- Role: Họ làm gì?
- Behavior: Họ dùng sản phẩm thế nào?
- Language: Họ mô tả sản phẩm bằng từ gì?

→ Output: **HXC Persona** = người yêu sản phẩm NHẤT

### Bước 3: ANALYZE

**Từ HXC** (câu 3 survey): "Cái gì HỌ YÊU?"
```
→ Feature/Value mà HXC nói nhiều nhất
→ DOUBLE DOWN — polish, extend, promote feature này
```

**Từ "On-the-fence"** (câu 4 survey): "Cái gì CẢN TRỞ họ yêu?"
```
→ Gap/Pain mà on-the-fence muốn cải thiện
→ FIX — convert họ thành HXC
```

### Bước 4: IMPLEMENT — Roadmap 50/50

```
PRODUCT ROADMAP:
├── 50% — Polish cái HXC YÊU
│   → Làm feature yêu thích TỐT HƠN NỮA
│   → Giữ HXC happy + tăng NPS
│
└── 50% — Fix cái on-the-fence CẦN
    → Giải quyết blocker
    → Convert on-the-fence → HXC
    → Tăng PMF score
```

**After 4-6 tuần: RE-SURVEY**

```
Round 1: 28% → identify HXC + gaps
Round 2: 35% → improved, but not yet
Round 3: 42% → ✅ PMF ACHIEVED!
```

## Key Insight

> **KHÔNG cần làm hài lòng TẤT CẢ users.**
> Chỉ cần: (1) giữ HXC happy, (2) convert on-the-fence.
> Ignore "Không thất vọng" — họ KHÔNG PHẢI target.
